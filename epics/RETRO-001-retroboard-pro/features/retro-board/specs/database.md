# Retro Board — Database Specification

## Overview

This document defines the complete database schema for the retro board feature. All tables use UUID primary keys generated by `gen_random_uuid()`. Timestamps are stored as `timestamptz` (UTC).

## Table Definitions

### boards

The central table for retro boards. Each board is tied to exactly one sprint.

```sql
CREATE TABLE boards (
  id            UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  sprint_id     UUID NOT NULL UNIQUE,
  template_id   UUID NOT NULL,
  phase         board_phase NOT NULL DEFAULT 'write',
  anonymous_mode BOOLEAN NOT NULL DEFAULT false,
  max_votes_per_user INTEGER NOT NULL DEFAULT 5,
  max_votes_per_card INTEGER NOT NULL DEFAULT 3,
  focus_item_id UUID,
  focus_item_type focus_item_type,
  created_by    UUID NOT NULL,
  created_at    TIMESTAMPTZ NOT NULL DEFAULT now(),
  updated_at    TIMESTAMPTZ NOT NULL DEFAULT now(),

  CONSTRAINT fk_boards_sprint
    FOREIGN KEY (sprint_id) REFERENCES sprints(id) ON DELETE CASCADE,
  CONSTRAINT fk_boards_template
    FOREIGN KEY (template_id) REFERENCES templates(id) ON DELETE RESTRICT,
  CONSTRAINT fk_boards_created_by
    FOREIGN KEY (created_by) REFERENCES users(id) ON DELETE RESTRICT,
  CONSTRAINT chk_boards_max_votes_per_user
    CHECK (max_votes_per_user BETWEEN 1 AND 99),
  CONSTRAINT chk_boards_max_votes_per_card
    CHECK (max_votes_per_card BETWEEN 1 AND 99),
  CONSTRAINT chk_boards_focus_consistency
    CHECK (
      (focus_item_id IS NULL AND focus_item_type IS NULL) OR
      (focus_item_id IS NOT NULL AND focus_item_type IS NOT NULL)
    )
);
```

**Columns:**

| Column | Type | Nullable | Default | Description |
|--------|------|----------|---------|-------------|
| id | UUID | No | gen_random_uuid() | Primary key |
| sprint_id | UUID | No | — | Sprint this board belongs to (unique 1:1) |
| template_id | UUID | No | — | Template used to create this board |
| phase | board_phase | No | 'write' | Current board phase |
| anonymous_mode | BOOLEAN | No | false | Whether card authors are hidden |
| max_votes_per_user | INTEGER | No | 5 | Maximum total votes per user on this board |
| max_votes_per_card | INTEGER | No | 3 | Maximum votes per user per card |
| focus_item_id | UUID | Yes | NULL | ID of the currently focused card or group |
| focus_item_type | focus_item_type | Yes | NULL | Type of the focused item (card or group) |
| created_by | UUID | No | — | User who created this board |
| created_at | TIMESTAMPTZ | No | now() | Creation timestamp |
| updated_at | TIMESTAMPTZ | No | now() | Last update timestamp |

**Constraints:**
- `sprint_id` is UNIQUE — enforces 1:1 relationship with sprints
- `template_id` uses ON DELETE RESTRICT — cannot delete a template in use
- `created_by` uses ON DELETE RESTRICT — cannot delete user who created boards
- `focus_item_id` and `focus_item_type` must both be NULL or both be non-NULL

### columns

Board columns define the lanes where cards are placed.

```sql
CREATE TABLE columns (
  id         UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  board_id   UUID NOT NULL,
  name       VARCHAR(100) NOT NULL,
  color      VARCHAR(7) NOT NULL DEFAULT '#6b7280',
  position   INTEGER NOT NULL DEFAULT 0,
  created_at TIMESTAMPTZ NOT NULL DEFAULT now(),

  CONSTRAINT fk_columns_board
    FOREIGN KEY (board_id) REFERENCES boards(id) ON DELETE CASCADE,
  CONSTRAINT chk_columns_color_hex
    CHECK (color ~ '^#[0-9a-fA-F]{6}$'),
  CONSTRAINT uq_columns_board_position
    UNIQUE (board_id, position) DEFERRABLE INITIALLY DEFERRED
);
```

**Columns:**

| Column | Type | Nullable | Default | Description |
|--------|------|----------|---------|-------------|
| id | UUID | No | gen_random_uuid() | Primary key |
| board_id | UUID | No | — | Board this column belongs to |
| name | VARCHAR(100) | No | — | Column display name |
| color | VARCHAR(7) | No | '#6b7280' | Hex color code for column header |
| position | INTEGER | No | 0 | Display order (0-indexed) |
| created_at | TIMESTAMPTZ | No | now() | Creation timestamp |

**Constraints:**
- Cascade delete when board is deleted
- Color must be valid hex format
- Position is unique per board (deferrable for reordering)

### cards

Individual feedback items on the board.

```sql
CREATE TABLE cards (
  id         UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  column_id  UUID NOT NULL,
  board_id   UUID NOT NULL,
  content    TEXT NOT NULL,
  author_id  UUID NOT NULL,
  position   INTEGER NOT NULL DEFAULT 0,
  created_at TIMESTAMPTZ NOT NULL DEFAULT now(),
  updated_at TIMESTAMPTZ NOT NULL DEFAULT now(),

  CONSTRAINT fk_cards_column
    FOREIGN KEY (column_id) REFERENCES columns(id) ON DELETE CASCADE,
  CONSTRAINT fk_cards_board
    FOREIGN KEY (board_id) REFERENCES boards(id) ON DELETE CASCADE,
  CONSTRAINT fk_cards_author
    FOREIGN KEY (author_id) REFERENCES users(id) ON DELETE RESTRICT,
  CONSTRAINT chk_cards_content_not_empty
    CHECK (length(trim(content)) > 0),
  CONSTRAINT chk_cards_content_max_length
    CHECK (length(content) <= 2000)
);
```

**Columns:**

| Column | Type | Nullable | Default | Description |
|--------|------|----------|---------|-------------|
| id | UUID | No | gen_random_uuid() | Primary key |
| column_id | UUID | No | — | Column this card is placed in |
| board_id | UUID | No | — | Board this card belongs to (denormalized for query performance) |
| content | TEXT | No | — | Card text content (1-2000 chars) |
| author_id | UUID | No | — | User who created this card |
| position | INTEGER | No | 0 | Display order within column (0-indexed) |
| created_at | TIMESTAMPTZ | No | now() | Creation timestamp |
| updated_at | TIMESTAMPTZ | No | now() | Last update timestamp |

**Design Notes:**
- `board_id` is denormalized from `column_id -> columns.board_id` for performance. It avoids a JOIN when querying all cards for a board.
- `author_id` is always stored even in anonymous mode — anonymous mode is enforced at the API layer, not the data layer.

### card_votes

Tracks individual votes. A user can cast multiple votes on the same card, up to limits.

```sql
CREATE TABLE card_votes (
  id          UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  card_id     UUID NOT NULL,
  user_id     UUID NOT NULL,
  vote_number INTEGER NOT NULL DEFAULT 1,
  created_at  TIMESTAMPTZ NOT NULL DEFAULT now(),

  CONSTRAINT fk_card_votes_card
    FOREIGN KEY (card_id) REFERENCES cards(id) ON DELETE CASCADE,
  CONSTRAINT fk_card_votes_user
    FOREIGN KEY (user_id) REFERENCES users(id) ON DELETE CASCADE,
  CONSTRAINT uq_card_votes_card_user_number
    UNIQUE (card_id, user_id, vote_number),
  CONSTRAINT chk_card_votes_vote_number_positive
    CHECK (vote_number >= 1)
);
```

**Columns:**

| Column | Type | Nullable | Default | Description |
|--------|------|----------|---------|-------------|
| id | UUID | No | gen_random_uuid() | Primary key |
| card_id | UUID | No | — | Card being voted on |
| user_id | UUID | No | — | User who cast this vote |
| vote_number | INTEGER | No | 1 | Sequence number for this user's votes on this card (1, 2, 3...) |
| created_at | TIMESTAMPTZ | No | now() | When the vote was cast |

**Design Notes:**
- `vote_number` allows a user to vote multiple times on the same card. Vote 1, vote 2, vote 3, etc.
- The UNIQUE constraint on `(card_id, user_id, vote_number)` prevents duplicate entries.
- Vote limit enforcement happens at the application level using the board's `max_votes_per_user` and `max_votes_per_card` settings.
- ON DELETE CASCADE on card ensures votes are cleaned up when a card is removed.

### card_groups

Named clusters of related cards, created during the grouping phase.

```sql
CREATE TABLE card_groups (
  id         UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  board_id   UUID NOT NULL,
  title      VARCHAR(200) NOT NULL,
  position   INTEGER NOT NULL DEFAULT 0,
  created_at TIMESTAMPTZ NOT NULL DEFAULT now(),

  CONSTRAINT fk_card_groups_board
    FOREIGN KEY (board_id) REFERENCES boards(id) ON DELETE CASCADE,
  CONSTRAINT chk_card_groups_title_not_empty
    CHECK (length(trim(title)) > 0)
);
```

**Columns:**

| Column | Type | Nullable | Default | Description |
|--------|------|----------|---------|-------------|
| id | UUID | No | gen_random_uuid() | Primary key |
| board_id | UUID | No | — | Board this group belongs to |
| title | VARCHAR(200) | No | — | Group display title |
| position | INTEGER | No | 0 | Display order among groups |
| created_at | TIMESTAMPTZ | No | now() | Creation timestamp |

### card_group_members

Join table linking cards to groups. A card can be in at most one group.

```sql
CREATE TABLE card_group_members (
  group_id UUID NOT NULL,
  card_id  UUID NOT NULL,

  CONSTRAINT pk_card_group_members
    PRIMARY KEY (group_id, card_id),
  CONSTRAINT fk_card_group_members_group
    FOREIGN KEY (group_id) REFERENCES card_groups(id) ON DELETE CASCADE,
  CONSTRAINT fk_card_group_members_card
    FOREIGN KEY (card_id) REFERENCES cards(id) ON DELETE CASCADE,
  CONSTRAINT uq_card_group_members_card
    UNIQUE (card_id)
);
```

**Columns:**

| Column | Type | Nullable | Description |
|--------|------|----------|-------------|
| group_id | UUID | No | Group the card belongs to |
| card_id | UUID | No | Card that is grouped |

**Design Notes:**
- Composite primary key `(group_id, card_id)` for the many-to-many relationship.
- The UNIQUE constraint on `card_id` ensures a card can only belong to one group at a time.
- ON DELETE CASCADE on both FKs means: deleting a group removes the membership rows; deleting a card removes its membership.

## Triggers

### Auto-update `updated_at`

```sql
CREATE OR REPLACE FUNCTION trigger_set_updated_at()
RETURNS TRIGGER AS $$
BEGIN
  NEW.updated_at = now();
  RETURN NEW;
END;
$$ LANGUAGE plpgsql;

CREATE TRIGGER set_boards_updated_at
  BEFORE UPDATE ON boards
  FOR EACH ROW
  EXECUTE FUNCTION trigger_set_updated_at();
```

### NOTIFY on board changes

Triggers that fire NOTIFY events for real-time synchronization.

```sql
CREATE OR REPLACE FUNCTION notify_board_change()
RETURNS TRIGGER AS $$
DECLARE
  board_uuid UUID;
  payload TEXT;
BEGIN
  -- Determine board_id based on which table was modified
  IF TG_TABLE_NAME = 'boards' THEN
    board_uuid := COALESCE(NEW.id, OLD.id);
    payload := json_build_object(
      'event', TG_ARGV[0],
      'board_id', board_uuid
    )::TEXT;
  ELSIF TG_TABLE_NAME = 'cards' THEN
    board_uuid := COALESCE(NEW.board_id, OLD.board_id);
    payload := json_build_object(
      'event', TG_ARGV[0],
      'board_id', board_uuid,
      'card_id', COALESCE(NEW.id, OLD.id),
      'column_id', COALESCE(NEW.column_id, OLD.column_id)
    )::TEXT;
  ELSIF TG_TABLE_NAME = 'card_votes' THEN
    -- Need to look up board_id from card
    SELECT c.board_id INTO board_uuid
    FROM cards c
    WHERE c.id = COALESCE(NEW.card_id, OLD.card_id);

    payload := json_build_object(
      'event', TG_ARGV[0],
      'board_id', board_uuid,
      'card_id', COALESCE(NEW.card_id, OLD.card_id)
    )::TEXT;
  ELSIF TG_TABLE_NAME = 'card_groups' THEN
    board_uuid := COALESCE(NEW.board_id, OLD.board_id);
    payload := json_build_object(
      'event', TG_ARGV[0],
      'board_id', board_uuid,
      'group_id', COALESCE(NEW.id, OLD.id)
    )::TEXT;
  END IF;

  PERFORM pg_notify('board_changes', payload);
  RETURN COALESCE(NEW, OLD);
END;
$$ LANGUAGE plpgsql;
```

```sql
-- Create NOTIFY trigger function
CREATE OR REPLACE FUNCTION notify_board_change()
RETURNS TRIGGER AS $$
DECLARE
  board_uuid UUID;
  payload TEXT;
BEGIN
  IF TG_TABLE_NAME = 'boards' THEN
    board_uuid := COALESCE(NEW.id, OLD.id);
    payload := json_build_object(
      'event', TG_ARGV[0],
      'board_id', board_uuid
    )::TEXT;
  ELSIF TG_TABLE_NAME = 'cards' THEN
    board_uuid := COALESCE(NEW.board_id, OLD.board_id);
    payload := json_build_object(
      'event', TG_ARGV[0],
      'board_id', board_uuid,
      'card_id', COALESCE(NEW.id, OLD.id),
      'column_id', COALESCE(NEW.column_id, OLD.column_id)
    )::TEXT;
  ELSIF TG_TABLE_NAME = 'card_votes' THEN
    SELECT c.board_id INTO board_uuid
    FROM cards c
    WHERE c.id = COALESCE(NEW.card_id, OLD.card_id);
    payload := json_build_object(
      'event', TG_ARGV[0],
      'board_id', board_uuid,
      'card_id', COALESCE(NEW.card_id, OLD.card_id)
    )::TEXT;
  ELSIF TG_TABLE_NAME = 'card_groups' THEN
    board_uuid := COALESCE(NEW.board_id, OLD.board_id);
    payload := json_build_object(
      'event', TG_ARGV[0],
      'board_id', board_uuid,
      'group_id', COALESCE(NEW.id, OLD.id)
    )::TEXT;
  END IF;

  PERFORM pg_notify('board_changes', payload);
  RETURN COALESCE(NEW, OLD);
END;
$$ LANGUAGE plpgsql;

CREATE TRIGGER notify_board_updated
  AFTER UPDATE ON boards
```

## Common Queries

### Get full board with all data

```sql
-- Fetch board with columns ordered by position
SELECT b.*,
       json_agg(
         json_build_object(
           'id', col.id,
           'name', col.name,
           'color', col.color,
           'position', col.position
         ) ORDER BY col.position
       ) AS columns
FROM boards b
JOIN columns col ON col.board_id = b.id
WHERE b.sprint_id = $1
GROUP BY b.id;

-- Fetch cards for a board with vote counts
SELECT c.*,
       u.name AS author_name,
       COALESCE(v.vote_count, 0) AS vote_count,
       COALESCE(uv.user_votes, 0) AS user_votes,
       cgm.group_id
FROM cards c
JOIN users u ON u.id = c.author_id
LEFT JOIN (
  SELECT card_id, COUNT(*) AS vote_count
  FROM card_votes
  GROUP BY card_id
) v ON v.card_id = c.id
LEFT JOIN (
  SELECT card_id, COUNT(*) AS user_votes
  FROM card_votes
  WHERE user_id = $2
  GROUP BY card_id
) uv ON uv.card_id = c.id
LEFT JOIN card_group_members cgm ON cgm.card_id = c.id
WHERE c.board_id = $3
ORDER BY c.column_id, c.position;

-- Fetch groups for a board with member card IDs
SELECT cg.*,
       COALESCE(
         array_agg(cgm.card_id ORDER BY cgm.card_id) FILTER (WHERE cgm.card_id IS NOT NULL),
         ARRAY[]::UUID[]
       ) AS card_ids,
       COALESCE(SUM(vc.cnt), 0) AS total_votes
FROM card_groups cg
LEFT JOIN card_group_members cgm ON cgm.group_id = cg.id
LEFT JOIN (
  SELECT card_id, COUNT(*) AS cnt
  FROM card_votes
  GROUP BY card_id
) vc ON vc.card_id = cgm.card_id
WHERE cg.board_id = $4
GROUP BY cg.id
ORDER BY cg.position;

-- Count votes by user on board
SELECT COUNT(*) AS total_votes_cast
FROM card_votes cv
JOIN cards c ON c.id = cv.card_id
WHERE c.board_id = $5
  AND cv.user_id = $6;
```

### Vote with limit enforcement (transactional)

```sql
-- Cast a vote with limit checks (run in a transaction)
BEGIN;

-- Lock the board row to prevent race conditions
SELECT max_votes_per_user, max_votes_per_card
FROM boards
WHERE id = $1
FOR UPDATE;

-- Check total votes by user on board
SELECT COUNT(*) AS total
FROM card_votes cv
JOIN cards c ON c.id = cv.card_id
WHERE c.board_id = $1 AND cv.user_id = $2;
-- If total >= max_votes_per_user, ROLLBACK and return error

-- Check votes by user on this specific card
SELECT COUNT(*) AS card_total
FROM card_votes
WHERE card_id = $3 AND user_id = $2;
-- If card_total >= max_votes_per_card, ROLLBACK and return error

-- Determine next vote number
SELECT COALESCE(MAX(vote_number), 0) + 1 AS next_vote
FROM card_votes
WHERE card_id = $3 AND user_id = $2;

-- Insert the vote
INSERT INTO card_votes (card_id, user_id, vote_number)
VALUES ($3, $2, next_vote);

COMMIT;
```

## Performance Considerations

1. **Denormalized `board_id` on cards**: Avoids JOIN through columns table when fetching all cards for a board.
2. **Composite indexes on `(board_id, column_id, position)`**: Optimizes the most common query pattern (get all cards for a board ordered by column and position).
3. **Deferrable unique constraint on column position**: Allows temporary duplicates during reorder operations within a transaction.
4. **Vote counting**: Aggregated at query time rather than stored as a counter to avoid race conditions. The indexes on `card_votes(card_id)` and `card_votes(card_id, user_id)` make these aggregations fast.
5. **NOTIFY payload size**: All payloads are kept under 100 bytes (event type + UUIDs only), well within PostgreSQL's 8KB limit.
