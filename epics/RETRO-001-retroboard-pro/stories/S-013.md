# S-013: Real-Time Card Sync via WebSocket

## User Story

As a **team member**, I want to **see card changes in real-time without refreshing the page** so that **the retro session feels collaborative and interactive**.

## Acceptance Criteria

- [x] All card operations (create, update, delete, move) are broadcast to connected board participants
- [x] Vote changes are broadcast in real-time to all participants
- [x] Group changes (create, update, dissolve, card assignment) are broadcast in real-time
- [x] Board settings changes are broadcast in real-time
- [x] WebSocket connection is authenticated using JWT token
- [x] Participants automatically reconnect on connection loss with exponential backoff
- [x] Missed events during disconnection are reconciled on reconnect (full board state fetch)
- [x] WebSocket events are scoped to the board room (only participants of that board receive events)
- [x] Server uses PostgreSQL LISTEN/NOTIFY to receive database change events
- [x] Latency for real-time updates is under 200ms on average

## Implementation Notes

- Use native WebSocket or a library like `ws` on the Hono server
- Implement room-based channels: `board:{boardId}`
- Event types: `card:created`, `card:updated`, `card:deleted`, `card:moved`, `vote:added`, `vote:removed`, `group:created`, `group:updated`, `group:dissolved`, `board:settings_updated`
- PostgreSQL LISTEN/NOTIFY on card/vote/group changes triggers WS broadcast
- Use a connection manager to track active connections per board room
