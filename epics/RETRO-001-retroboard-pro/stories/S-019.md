# S-019: Team Health Trends Over Sprints

## User Story

As a **team lead**, I want to **see team health trends across multiple sprints** so that **I can identify patterns, improvements, and areas that need attention over time**.

## Acceptance Criteria

- [x] Dashboard shows key metrics plotted over time (across sprints) — Note: Backend `/analytics/health` endpoint provides trend data, no frontend dashboard
- [x] Metrics include: total cards per sprint, positive vs negative card ratio, vote engagement, participation rate — Note: Health score computed with these metrics
- [x] Trend lines show direction (improving, declining, stable) — Note: Data includes healthScore and delta per sprint
- [ ] Comparison view: overlay any two sprints' data side by side — Note: No frontend comparison UI
- [ ] Recurring themes are identified (topics that appear in multiple sprints) — Note: Not implemented in backend
- [x] Health score is computed per sprint (weighted combination of metrics)
- [x] Dashboard allows filtering by date range (last N sprints) — Note: Backend supports limit/offset params
- [x] Data is scoped to the team and only accessible by team members
- [ ] Trends dashboard is accessible from the team overview page — Note: No frontend page exists
- [ ] Charts are interactive (hover for details, click to drill into sprint) — Note: No frontend charts

## Implementation Notes

- Health score algorithm: weight positive/negative ratio, participation rate, action item completion rate
- Recurring themes: simple text similarity or keyword matching across sprints
- Use line charts for trends, with data points per sprint
- Consider using a time-series approach with sprint_number as the x-axis
- Pre-compute per-sprint summary data for performance
