# S-019: Team Health Trends Over Sprints

## User Story

As a **team lead**, I want to **see team health trends across multiple sprints** so that **I can identify patterns, improvements, and areas that need attention over time**.

## Acceptance Criteria

- [ ] Dashboard shows key metrics plotted over time (across sprints)
- [ ] Metrics include: total cards per sprint, positive vs negative card ratio, vote engagement, participation rate
- [ ] Trend lines show direction (improving, declining, stable)
- [ ] Comparison view: overlay any two sprints' data side by side
- [ ] Recurring themes are identified (topics that appear in multiple sprints)
- [ ] Health score is computed per sprint (weighted combination of metrics)
- [ ] Dashboard allows filtering by date range (last N sprints)
- [ ] Data is scoped to the team and only accessible by team members
- [ ] Trends dashboard is accessible from the team overview page
- [ ] Charts are interactive (hover for details, click to drill into sprint)

## Implementation Notes

- Health score algorithm: weight positive/negative ratio, participation rate, action item completion rate
- Recurring themes: simple text similarity or keyword matching across sprints
- Use line charts for trends, with data points per sprint
- Consider using a time-series approach with sprint_number as the x-axis
- Pre-compute per-sprint summary data for performance
