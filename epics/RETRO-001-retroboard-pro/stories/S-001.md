# S-001: User Registration & Login

## User Story

As a **user**, I want to **create an account and log in** so that **I can access retro boards and collaborate with my team**.

## Acceptance Criteria

- [ ] User can register with email, password, and display_name fields
- [ ] Password must be at least 8 characters with at least one uppercase letter, one lowercase letter, and one number
- [ ] Email must be a valid email format and is stored in lowercase
- [ ] Duplicate email registration returns a 409 Conflict error with a clear message
- [ ] Successful registration returns a 201 response with user profile (excluding password)
- [ ] Login with valid email/password returns JWT access token and refresh token
- [ ] Login with invalid credentials returns a 401 Unauthorized error
- [ ] Validation errors return 422 with field-level error messages
- [ ] Passwords are hashed using bcrypt with a cost factor of 12 before storage
- [ ] Display name is trimmed and must be between 2 and 50 characters
- [ ] Email verification status is tracked (defaults to unverified)

## Implementation Notes

- Use Hono framework for API routing
- Use `bcrypt` for password hashing
- Use `jose` or `jsonwebtoken` for JWT token generation
- Store users in PostgreSQL `users` table
- Access token TTL: 15 minutes; Refresh token TTL: 7 days
- All auth endpoints are under `/api/v1/auth/*`
