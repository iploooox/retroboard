# S-011: Group Related Cards into Clusters

## User Story

As a **facilitator**, I want to **group related cards into clusters** so that **similar feedback is consolidated and discussions are more focused**.

## Acceptance Criteria

- [ ] Facilitator can create a named group/cluster within a column
- [ ] Facilitator can drag cards into a group
- [ ] A group displays its member cards in a stacked or nested view
- [ ] Group shows the total vote count of all its member cards
- [ ] Group name is editable by the facilitator
- [ ] Cards can be removed from a group back to the column
- [ ] A group can be dissolved, returning all cards to the column
- [ ] Groups persist across page refreshes
- [ ] Any team member can suggest a grouping (facilitator approves in facilitated mode)
- [ ] Groups are visually distinct from ungrouped cards (border, background color, indent)

## Implementation Notes

- `card_groups` table: id, board_id, column_id, name, sort_order, created_by, created_at, updated_at
- Add `group_id` nullable FK to `cards` table
- Cards in a group maintain their own sort_order within the group
- Group vote count is a computed aggregation of member cards
- Dissolving a group: set group_id to null on all member cards, delete the group record
