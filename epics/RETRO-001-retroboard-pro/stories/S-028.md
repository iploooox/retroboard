# S-028: Icebreaker Generator

## User Story

As a **facilitator**, I want to **generate a random icebreaker question to start the retro** so that **the team warms up, feels comfortable, and the retrospective begins on a positive note**.

## Acceptance Criteria

- [x] Facilitator can generate a random icebreaker question from a curated library — GET /icebreakers/random endpoint exists
- [x] Icebreaker library contains at least 50 questions across categories — 55 questions seeded in migration 021
- [x] Categories include: Fun, Team-Building, Reflective, Creative, Quick — all 5 categories implemented
- [x] Facilitator can filter by category — GET /icebreakers/random?category=X supported, UI has category filter buttons
- [x] Facilitator can skip/re-roll to get a different question — "New Question" button implemented
- [x] Selected icebreaker is displayed to all board participants in real-time — WebSocket broadcast implemented, all participants see same question when facilitator refreshes
- [x] Facilitator can dismiss the icebreaker display when done — dismiss button (X) implemented
- [x] Icebreaker history is tracked per team (avoid repeating recent ones) — team_icebreaker_history table + service logic
- [ ] Custom icebreaker questions can be added by team admin — backend POST endpoint exists but **no frontend UI** for adding custom questions
- [x] Icebreaker feature is optional and can be skipped — can be dismissed anytime

## Implementation Notes

- `icebreakers` table: id, question, category, is_system, created_by, created_at
- `team_icebreaker_history` table: id, team_id, icebreaker_id, used_at
- Seed 50+ system icebreaker questions across categories
- Random selection: exclude recently used (last 10) for the team
- Display via WebSocket broadcast: `icebreaker:shown`, `icebreaker:dismissed`

## Implementation Details (Task #2)

**Completed:** Real-time WebSocket broadcast of icebreaker questions

**Backend:**
- Updated `GET /icebreakers/random` route to accept optional `boardId` query parameter
- Added import of `broadcastToBoard` function from `ws/index.ts`
- After fetching random icebreaker, if `boardId` is provided, broadcast to all board participants:
  - Event type: `'icebreaker_update'`
  - Payload: `{ question, category, id }`
- Uses existing WebSocket infrastructure (`roomManager.broadcast`)

**Frontend:**
- Updated `IcebreakerCard` component:
  - Added `boardId` prop to interface
  - Updated API calls to include `boardId` query parameter
  - Added WebSocket listener for `'icebreaker_update'` events
  - When event received, updates local icebreaker state with new question
  - Listener properly cleans up on unmount
- Updated `BoardPage.tsx` to pass `board.id` as `boardId` prop to `IcebreakerCard`

**Result:**
- When facilitator clicks "New Question" or changes category, all participants see the same icebreaker question in real-time
- No need for participants to manually refresh
- Works seamlessly with existing WebSocket connection for board
