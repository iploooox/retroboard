# S-009: Anonymous and Named Card Modes

## User Story

As a **facilitator**, I want to **toggle between anonymous and named card modes** so that **team members feel comfortable sharing honest feedback when anonymity is needed**.

## Acceptance Criteria

- [x] Board has an `anonymous_mode` setting (boolean) configured at creation or updated by facilitator
- [x] When anonymous mode is ON, all cards display "Anonymous" instead of the author's name
- [x] When anonymous mode is OFF, cards display the author's display name and avatar
- [x] Facilitator can toggle anonymous mode at any time during the retro
- [x] Mode change applies immediately to all existing and new cards on the board
- [x] The author can always see their own cards are theirs (subtle indicator like "You")
- [x] API never returns author information to non-author users when anonymous mode is ON
- [x] Cards store the author_id regardless of mode (for internal tracking/analytics)
- [x] Anonymous mode state is visible to all participants (icon/badge on board header)
- [x] Mode toggle sends a real-time notification to all connected participants

## Implementation Notes

- Anonymous mode is stored in `boards.settings` JSONB field
- When anonymous mode is ON, the API serializer should strip author details for non-owners
- The `is_anonymous` flag on individual cards can override the board setting for mixed mode (future)
- Real-time mode change event: `board:settings_updated`
