# S-002: Session Management with JWT

## User Story

As a **logged-in user**, I want to **have my session automatically managed with token refresh** so that **I don't get unexpectedly logged out during a retro session**.

## Acceptance Criteria

- [ ] Access tokens expire after 15 minutes
- [ ] Refresh tokens expire after 7 days and are stored in the database
- [ ] Client automatically refreshes access token before expiry using the refresh token
- [ ] `POST /api/v1/auth/refresh` accepts a refresh token and returns a new access/refresh token pair
- [ ] Old refresh token is invalidated after use (rotation)
- [ ] `POST /api/v1/auth/logout` invalidates the current refresh token
- [ ] Using an invalidated or expired refresh token returns 401 and forces re-login
- [ ] Multiple devices/sessions are supported (each has its own refresh token)
- [ ] Auth middleware extracts and validates JWT from Authorization Bearer header
- [ ] Protected routes return 401 if no valid token is provided

## Implementation Notes

- Store refresh tokens in a `refresh_tokens` table with user_id, token_hash, expires_at, revoked_at
- Use token rotation: each refresh issues a new refresh token and invalidates the old one
- Detect refresh token reuse (potential theft) by checking if a revoked token is used
- Auto-refresh on the frontend should trigger when access token has less than 2 minutes remaining
