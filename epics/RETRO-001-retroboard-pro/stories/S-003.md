# S-003: Create and Manage Teams

## User Story

As a **user**, I want to **create and manage teams** so that **I can organize retrospectives by team and collaborate with my colleagues**.

## Acceptance Criteria

- [x] Authenticated user can create a team with a name and optional description
- [x] Team creator is automatically assigned the "admin" role
- [x] Team names must be between 2 and 100 characters
- [x] User can view a list of all teams they belong to
- [x] Team admin can update team name and description
- [x] Team admin can delete a team (soft delete)
- [x] Deleting a team archives all associated sprints and boards
- [x] User can view team details including member list and their roles
- [x] API returns proper 403 for unauthorized team operations
- [x] Teams have a unique slug generated from the name for URL-friendly access

## Implementation Notes

- Teams table with id, name, slug, description, created_by, created_at, updated_at, deleted_at
- Team memberships managed via `team_members` join table
- Slug generation: lowercase, replace spaces with hyphens, append short random suffix if collision
- Use soft deletes for teams (set deleted_at timestamp)
