# S-016: Built-In Countdown Timer per Phase

## User Story

As a **facilitator**, I want to **set a countdown timer for each retro phase** so that **the team stays focused and the retrospective fits within the allocated time**.

## Acceptance Criteria

- [ ] Facilitator can start a countdown timer with a configurable duration (1-30 minutes)
- [ ] Timer is visible to all participants in real-time, synchronized across all screens
- [ ] Timer counts down and displays minutes:seconds format
- [ ] Facilitator can pause, resume, and reset the timer
- [ ] When the timer expires, an audible notification and visual alert are shown to all participants
- [ ] Each phase can have a pre-configured default timer duration in board settings
- [ ] Facilitator can extend the timer by 1, 2, or 5 minutes
- [ ] Timer state persists through brief disconnections (server is the source of truth)
- [ ] Timer is optional -- phases can run without a timer
- [ ] Multiple timers cannot run simultaneously on the same board

## Implementation Notes

- Timer state: { startedAt, duration, pausedAt, remainingOnPause } stored in-memory on server
- Server broadcasts timer events: `timer:started`, `timer:paused`, `timer:resumed`, `timer:reset`, `timer:expired`, `timer:extended`
- Client calculates remaining time from server's startedAt + duration (avoids clock sync issues)
- Use `setInterval` on client with server reconciliation every 10 seconds
- Default durations per phase stored in board settings JSONB
