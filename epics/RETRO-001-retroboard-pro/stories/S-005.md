# S-005: Team Roles and Permission Enforcement

## User Story

As a **team admin**, I want to **assign roles (admin, facilitator, member) to team members** so that **appropriate permissions are enforced and sensitive operations are restricted**.

## Acceptance Criteria

- [ ] Three roles exist: admin, facilitator, member (ordered by privilege level)
- [ ] Admin can: manage team settings, manage members, manage roles, delete team, all facilitator permissions
- [ ] Facilitator can: create/manage sprints, create/manage boards, control facilitation phases, all member permissions
- [ ] Member can: participate in boards, add/edit own cards, vote, view team data
- [ ] Team admin can change any member's role
- [ ] Team must always have at least one admin (cannot demote last admin)
- [ ] Team admin can remove members from the team
- [ ] Permission checks are enforced at the API middleware level
- [ ] Role changes take effect immediately (no cached permissions)
- [ ] API returns 403 Forbidden with a clear message when permission is denied

## Implementation Notes

- Roles stored in `team_members.role` as enum: 'admin', 'facilitator', 'member'
- Create a permissions map/matrix that maps roles to allowed actions
- Middleware: `requireTeamRole('admin')`, `requireTeamRole('facilitator')`, etc.
- Permission checks should be composable and reusable across routes
