# S-008: Add, Edit, and Delete Cards in Columns

## User Story

As a **team member**, I want to **add, edit, and delete cards in board columns** so that **I can share my feedback and thoughts during the retrospective**.

## Acceptance Criteria

- [ ] Team member can add a card to any column on an active board
- [ ] Card content supports plain text with a maximum of 500 characters
- [ ] Card displays the author's display name (unless anonymous mode is on)
- [ ] Card author can edit their own card content
- [ ] Card author can delete their own card
- [ ] Facilitator or admin can delete any card
- [ ] Cards are displayed in creation order within their column (newest at bottom)
- [ ] Cards can be reordered within a column via drag-and-drop (sort_order updates)
- [ ] Cards can be moved between columns via drag-and-drop
- [ ] Empty card content is not allowed (whitespace-only rejected)
- [ ] Optimistic UI updates with rollback on server error

## Implementation Notes

- `cards` table: id, board_id, column_id, author_id, content, is_anonymous, sort_order, created_at, updated_at
- Sort order: use fractional indexing or gap-based ordering to avoid rewriting all orders on move
- Card mutations should trigger WebSocket events (implemented in S-013)
- Validate board is in active status before allowing card operations
