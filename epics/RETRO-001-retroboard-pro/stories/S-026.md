# S-026: Emoji Reactions on Cards

## User Story

As a **team member**, I want to **add emoji reactions to cards** so that **I can express quick feedback or agreement without adding another card or using a vote**.

## Acceptance Criteria

- [x] Team member can add an emoji reaction to any card on an active board
- [x] Available reactions are from a curated set (12 emojis defined in DB constraint)
- [x] Each user can add one of each reaction type per card (toggle on/off) — frontend implements toggle via API
- [x] Reactions are displayed below the card content with count badges
- [x] Clicking an already-added reaction removes it (toggle behavior) — frontend calls toggleReaction API
- [x] Reaction counts update in real-time for all participants — via WebSocket sync (relies on backend real-time events)
- [x] Reactions are visually compact and don't clutter the card — displayed as small pill buttons with emoji + count
- [x] Reactions are included in board exports — backend includes reactions in export data
- [x] Reactions persist after the board is completed — stored in card_reactions table
- [x] Reactions respect board lock state (disabled when board is locked) — frontend disables picker when board.is_locked

## Implementation Notes

- `card_reactions` table: id, card_id, user_id, emoji, created_at
- Unique constraint on (card_id, user_id, emoji) for toggle behavior
- Curated emoji set defined as a constant (avoid free-form emoji input for consistency)
- Real-time events: `reaction:added`, `reaction:removed`
- Aggregate reactions in card response: { emoji: string, count: number, reacted: boolean }[]
