# S-026: Emoji Reactions on Cards

## User Story

As a **team member**, I want to **add emoji reactions to cards** so that **I can express quick feedback or agreement without adding another card or using a vote**.

## Acceptance Criteria

- [ ] Team member can add an emoji reaction to any card on an active board
- [ ] Available reactions are from a curated set (thumbs up, thumbs down, heart, fire, thinking, laughing, 100, eyes)
- [ ] Each user can add one of each reaction type per card (toggle on/off)
- [ ] Reactions are displayed below the card content with count badges
- [ ] Clicking an already-added reaction removes it (toggle behavior)
- [ ] Reaction counts update in real-time for all participants
- [ ] Reactions are visually compact and don't clutter the card
- [ ] Reactions are included in board exports
- [ ] Reactions persist after the board is completed
- [ ] Reactions respect board lock state (disabled when board is locked)

## Implementation Notes

- `card_reactions` table: id, card_id, user_id, emoji, created_at
- Unique constraint on (card_id, user_id, emoji) for toggle behavior
- Curated emoji set defined as a constant (avoid free-form emoji input for consistency)
- Real-time events: `reaction:added`, `reaction:removed`
- Aggregate reactions in card response: { emoji: string, count: number, reacted: boolean }[]
