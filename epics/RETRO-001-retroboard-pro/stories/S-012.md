# S-012: Basic Templates (WWW/Delta, Start/Stop/Continue)

## User Story

As a **facilitator**, I want to **choose from basic retro templates** so that **I can quickly set up a board with a proven format without manual column configuration**.

## Acceptance Criteria

- [x] System provides at least two built-in templates: "What Went Well / Delta" and "Start / Stop / Continue"
- [x] Each template defines a set of columns with names, descriptions, colors, and sort order
- [x] Templates are read-only system data (not editable by users)
- [x] Template list is available via API for board creation selection
- [x] Each template includes a brief description and recommended use case
- [x] Templates are seeded into the database on initial setup / migration
- [x] WWW/Delta template has columns: "What Went Well" (green), "Delta / Changes" (orange)
- [x] Start/Stop/Continue template has columns: "Start" (green), "Stop" (red), "Continue" (blue)
- [x] Templates can be extended with new ones via database seed scripts

## Implementation Notes

- `templates` table: id, name, description, is_system, created_by, created_at
- `template_columns` table: id, template_id, name, description, color, sort_order
- System templates have `is_system = true` and `created_by = null`
- Seed script runs on database migration to insert/update system templates
- Template data is relatively static; consider caching in the API layer
