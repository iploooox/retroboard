# S-021: Sentiment Analysis on Cards

## User Story

As a **team lead**, I want to **see sentiment analysis of retro cards** so that **I can quickly gauge the team's overall mood and identify areas of concern or celebration**.

## Acceptance Criteria

- [x] Each card is assigned a sentiment score: positive, neutral, or negative
- [x] Sentiment is computed automatically based on card text content
- [x] Sprint-level sentiment summary shows distribution (% positive, neutral, negative)
- [x] Per-column sentiment breakdown is available (e.g., "What Went Well" is 80% positive)
- [x] Sentiment trend chart across sprints shows mood changes over time
- [x] Sentiment labels are displayed subtly on cards (colored dot or emoji)
- [x] Sentiment analysis uses a lexicon-based approach (no external API dependency)
- [x] Custom sentiment words can be added to the lexicon by team admin
- [x] Sentiment score is computed on card creation and updated on card edit
- [x] Analysis works for English text (with future extensibility for other languages)

## Implementation Notes

- Use a pre-built sentiment lexicon (AFINN, SentiWordStrength, or custom)
- Seed the lexicon into a `sentiment_lexicon` table: word, score (-5 to +5)
- Compute sentiment at the card level: sum word scores, normalize to positive/neutral/negative
- Store computed sentiment on the card record for fast retrieval
- Trigger recomputation on card content update
- Consider using PostgreSQL text search for tokenization

  - `DELETE /api/v1/teams/:teamId/sentiment/lexicon/:word` - Delete word (admin/facilitator only)
