# S-021: Sentiment Analysis on Cards

## User Story

As a **team lead**, I want to **see sentiment analysis of retro cards** so that **I can quickly gauge the team's overall mood and identify areas of concern or celebration**.

## Acceptance Criteria

- [x] Each card is assigned a sentiment score: positive, neutral, or negative — Note: Calculated via SQL function, not stored on cards
- [x] Sentiment is computed automatically based on card text content — Note: calculate_card_sentiment() function in migration 016
- [x] Sprint-level sentiment summary shows distribution (% positive, neutral, negative) — Note: Backend `/analytics/sentiment` endpoint provides this
- [ ] Per-column sentiment breakdown is available (e.g., "What Went Well" is 80% positive) — Note: Not implemented
- [x] Sentiment trend chart across sprints shows mood changes over time — Note: Backend endpoint provides trend data, no frontend chart
- [ ] Sentiment labels are displayed subtly on cards (colored dot or emoji) — Note: No frontend implementation
- [x] Sentiment analysis uses a lexicon-based approach (no external API dependency) — Note: sentiment_lexicon table with AFINN-based scoring
- [ ] Custom sentiment words can be added to the lexicon by team admin — Note: No admin UI for lexicon management
- [x] Sentiment score is computed on card creation and updated on card edit — Note: Calculated on-the-fly via SQL function
- [x] Analysis works for English text (with future extensibility for other languages)

## Implementation Notes

- Use a pre-built sentiment lexicon (AFINN, SentiWordStrength, or custom)
- Seed the lexicon into a `sentiment_lexicon` table: word, score (-5 to +5)
- Compute sentiment at the card level: sum word scores, normalize to positive/neutral/negative
- Store computed sentiment on the card record for fast retrieval
- Trigger recomputation on card content update
- Consider using PostgreSQL text search for tokenization
