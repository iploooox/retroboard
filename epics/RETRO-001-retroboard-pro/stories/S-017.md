# S-017: Facilitator Controls (Lock Board, Reveal Cards, Next Phase)

## User Story

As a **facilitator**, I want to **have dedicated controls to lock the board, reveal hidden cards, and advance phases** so that **I can effectively guide the retrospective session**.

## Acceptance Criteria

- [x] Facilitator can lock the board, preventing all card creation, editing, voting, and grouping
- [x] Board lock state is broadcast in real-time and enforced server-side
- [x] When locked, participants see a clear "Board Locked" indicator and all input controls are disabled
- [x] Facilitator can unlock the board to resume normal operations
- [x] During "write" phase with hidden cards, facilitator can reveal all cards at once
- [ ] Card reveal is animated (cards flip or fade in) for all participants simultaneously — Note: Cards reveal instantly without animation
- [x] Facilitator has a control panel/toolbar that is only visible to users with facilitator or admin role
- [x] Facilitator can highlight a specific card for discussion (card spotlight)
- [x] Spotlighted card is visually emphasized on all participants' screens
- [x] Facilitator can clear the spotlight to return to normal view

## Implementation Notes

- Board lock state stored in boards.settings JSONB: `{ is_locked: boolean }`
- Card visibility: add `is_revealed` boolean to board settings (applies to write phase)
- Spotlight: `board:card_spotlighted` event with cardId or null to clear
- Facilitator controls rendered in a floating toolbar anchored to the bottom of the board
- All facilitator actions are validated server-side (role check middleware)
