# S-014: Live Cursors and Presence Indicators

## User Story

As a **team member**, I want to **see who is currently on the board and where their cursors are** so that **I feel the presence of my teammates and can follow their focus**.

## Acceptance Criteria

- [x] When a user joins a board, their avatar and name appear in a "participants" bar
- [x] Participant list shows online/idle/offline status for each connected user
- [x] User's cursor position is broadcast to other participants in real-time
- [x] Each participant's cursor is rendered on other screens with their name label and a unique color
- [x] Cursor updates are throttled to avoid excessive network traffic (max 20 updates/second)
- [x] When a user disconnects, their cursor fades out after 3 seconds and is removed
- [ ] Idle detection: user is marked idle after 2 minutes of no mouse/keyboard activity — server-side presence has no idle timeout
- [x] Participant count is shown in the board header
- [x] Presence state persists across brief disconnections (reconnect within 10 seconds keeps status)

## Implementation Notes

- Cursor data: { userId, x, y, boardId, timestamp }
- Send cursor position via WebSocket, not REST API
- Use requestAnimationFrame for smooth cursor rendering on receiving end
- Throttle outgoing cursor events using requestAnimationFrame or lodash throttle
- Presence stored in-memory on the server (not persisted to DB)
- Color assignment: hash userId to a color from a preset palette
