# S-029: Onboarding Flow for New Teams

## User Story

As a **new user**, I want to **be guided through setting up my first team and retro board** so that **I can quickly understand how to use RetroBoard Pro and start running retrospectives**.

## Acceptance Criteria

- [ ] First-time users see an onboarding wizard after registration
- [ ] Onboarding steps: Welcome -> Create Team -> Invite Members -> Create Sprint -> Start First Retro
- [ ] Each step has clear instructions and visual guidance
- [ ] Users can skip any step and complete it later
- [x] Onboarding progress is tracked and persisted (user can resume where they left off) — onboarding_data JSONB column + service
- [ ] Skipped steps appear as suggestions in the dashboard until completed
- [ ] Onboarding includes tooltips/pointers on key UI elements (guided tour)
- [x] Onboarding is only shown once per user (can be re-triggered from settings) — onboarding_completed_at + reset endpoint
- [ ] Interactive demo board option: pre-populated board to explore features
- [ ] Onboarding completion triggers a celebratory animation/message

## Implementation Notes

- Add `onboarding_completed_at` and `onboarding_step` columns to users table
- Store onboarding progress: { current_step: number, completed_steps: string[], skipped_steps: string[] }
- Demo board: create a special read-only board with sample data for exploration
- Use a tooltip/spotlight library (e.g., react-joyride, shepherd.js) for guided tour
- Onboarding state persisted server-side for cross-device consistency
