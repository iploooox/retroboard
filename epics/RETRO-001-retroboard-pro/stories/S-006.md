# S-006: Sprint CRUD with Date Ranges and Status Management

## User Story

As a **facilitator**, I want to **create and manage sprints for my team** so that **retrospectives are organized by sprint cycle and progress can be tracked over time**.

## Acceptance Criteria

- [ ] Facilitator or admin can create a sprint with name, start_date, end_date, and optional goal
- [ ] Sprint name must be between 2 and 100 characters
- [ ] Start date must be before end date
- [ ] Sprint statuses: planned, active, completed
- [ ] Only one sprint per team can be in "active" status at a time
- [ ] Facilitator can update sprint details (name, dates, goal, status)
- [ ] Facilitator can transition sprint status: planned -> active -> completed
- [ ] Completed sprints cannot be modified (except by admin)
- [ ] User can view all sprints for a team, sorted by start date (newest first)
- [ ] Sprint detail view shows associated retro boards and action items
- [ ] Sprints are scoped to a team (team members only can access)

## Implementation Notes

- Store sprints in `sprints` table: id, team_id, name, goal, start_date, end_date, status, created_by, created_at, updated_at
- Status transitions are validated server-side (no skipping states)
- When activating a sprint, auto-complete any previously active sprint for the same team
- Consider adding sprint number auto-increment per team
