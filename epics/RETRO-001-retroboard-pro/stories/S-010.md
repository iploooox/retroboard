# S-010: Vote on Cards with Configurable Vote Limits

## User Story

As a **team member**, I want to **vote on cards to indicate which topics are most important** so that **the team can prioritize discussion on high-impact items**.

## Acceptance Criteria

- [x] Team member can vote on any card on an active board
- [x] Each vote increments the card's vote count by 1
- [x] User can remove their own vote from a card
- [x] User can vote on the same card multiple times (up to the per-card limit, default: 1)
- [x] Board has a configurable `max_votes_per_user` setting (default: 5)
- [x] User cannot exceed their total vote limit for the board
- [x] Vote counts are displayed on each card
- [ ] Cards can be sorted by vote count within a column (facilitator control) — not yet implemented as FE sort control
- [x] Voting is only allowed during the "vote" facilitation phase (when phases are active)
- [x] Real-time vote count updates are broadcast to all participants

## Implementation Notes

- `votes` table: id, card_id, user_id, created_at
- Enforce vote limits at the service layer: count user's votes across the board
- Per-card vote limit and total board vote limit are both configurable in board settings
- Default settings: max_votes_per_user: 5, max_votes_per_card_per_user: 1
- Vote operations should be idempotent where possible
