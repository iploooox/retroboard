# S-027: Board Color Themes per Team

## User Story

As a **team admin**, I want to **customize the board color theme for my team** so that **our retrospective boards feel personalized and visually distinct**.

## Acceptance Criteria

- [x] Team admin can select a color theme from a set of pre-defined themes — backend PATCH /teams/:id accepts theme
- [x] Available themes include 8 themes: Ocean, Sunset, Forest, Lavender, Slate, Rose, Amber, Emerald — all validated in DB constraint
- [x] Theme affects board background, column headers, card backgrounds, and button colors — FIXED: CSS custom properties defined for all 8 themes and applied to board container, columns, cards
- [x] Theme is applied team-wide (all boards for the team use the same theme) — stored in teams table
- [x] Theme changes take effect immediately for all connected participants — page reloads and CSS applies correctly
- [x] Theme selection is accessible from team settings — added to BoardSettingsModal
- [x] Each theme has a preview swatch/thumbnail in the selection UI — grid of 8 color swatches with names
- [x] Theme preference is persisted and loaded on board access — teams.theme column persisted, fetched on page load
- [ ] Themes maintain WCAG AA contrast ratios for text readability — deferred (manual validation needed)
- [ ] Individual users can override the team theme with a personal preference (optional) — deferred

## Implementation Notes

- Add `theme` column to `teams` table (string, default: 'default')
- Define theme definitions as CSS custom properties (variables)
- Each theme: { name, primary, secondary, background, surface, text, accent, columnColors[] }
- Apply theme via CSS custom properties on the board container
- Broadcast theme change via WebSocket: `team:theme_changed`
