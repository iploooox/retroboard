# S-007: Create Retro Board for a Sprint with Template Selection

## User Story

As a **facilitator**, I want to **create a retro board for a sprint and select a template** so that **the board is pre-configured with appropriate columns for the retrospective format**.

## Acceptance Criteria

- [ ] Facilitator can create a retro board linked to a specific sprint
- [ ] Board creation requires selecting a template (e.g., WWW/Delta, Start/Stop/Continue)
- [ ] Selected template pre-populates the board with the correct columns and configuration
- [ ] Board has a name (defaults to "Sprint {N} Retro" but is editable)
- [ ] Each sprint can have multiple retro boards (but typically one)
- [ ] Board statuses: draft, active, completed, archived
- [ ] Board settings include: anonymous mode toggle, vote limit per user, timer duration
- [ ] Facilitator can update board settings before and during the retro
- [ ] Team members can view a list of boards for the current sprint
- [ ] Board detail view shows all columns as defined by the template

## Implementation Notes

- `boards` table: id, sprint_id, team_id, name, template_id, status, settings (JSONB), created_by, created_at, updated_at
- `board_columns` table: id, board_id, name, description, color, sort_order, created_at
- Settings JSONB: { anonymous_mode: boolean, max_votes_per_user: number, timer_duration_seconds: number }
- On board creation, copy template columns into board_columns (denormalized for customization)
