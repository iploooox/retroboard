# S-023: Action Item Carry-Over Between Sprints

## User Story

As a **facilitator**, I want to **carry over incomplete action items to the next sprint** so that **unfinished improvements are not forgotten and remain visible to the team**.

## Acceptance Criteria

- [x] When creating a new retro board, facilitator is prompted to review incomplete action items from previous sprints
- [x] Facilitator can select which incomplete action items to carry over to the new sprint
- [x] Carried-over action items retain their original title, description, and assignee
- [x] Carried-over items get a new link to the current board/sprint while preserving history
- [x] A "carry-over count" badge indicates how many times an action item has been carried over
- [x] Carried-over items are visually distinct on the board (e.g., different border or icon)
- [x] Team can view action item history across sprints (where it originated, how many carry-overs)
- [x] Facilitator can bulk-carry-over all incomplete items with one click
- [x] Action items that are carried over more than 3 times are flagged for attention
- [x] Carry-over is logged in action item history for traceability

## Implementation Notes

- Add `original_action_item_id` and `carry_over_count` columns to action_items table
- Carry-over creates a new action item record linked to the original via `original_action_item_id`
- Increment `carry_over_count` on each carry-over
- Original action item status is set to "carried_over" (new status)
- Action item history: query chain of original_action_item_id references
