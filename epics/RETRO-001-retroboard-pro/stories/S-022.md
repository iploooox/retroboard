# S-022: Action Items with Assignee and Due Date

## User Story

As a **team member**, I want to **create action items from retro discussions with an assignee and due date** so that **feedback leads to concrete improvements that are tracked to completion**.

## Acceptance Criteria

- [ ] Any team member can create an action item during the retro
- [ ] Action item has: title, description (optional), assignee (team member), due date, status
- [ ] Action item statuses: open, in_progress, completed
- [ ] Action items can be linked to a specific card or group (optional)
- [ ] Assignee must be a member of the team
- [ ] Due date is optional but encouraged (default suggestion: next sprint's end date)
- [ ] Action items are listed in a dedicated panel on the board
- [ ] Assignee can update status of their action items
- [ ] Facilitator or admin can update any action item
- [ ] Action items are also accessible from the sprint detail page
- [ ] Completed action items show completion date

## Implementation Notes

- `action_items` table: id, board_id, sprint_id, team_id, card_id (nullable), group_id (nullable), title, description, assignee_id, due_date, status, completed_at, created_by, created_at, updated_at
- Action items are scoped to a board but also queryable by sprint and team
- Status transitions: open -> in_progress -> completed (also open -> completed directly)
- Real-time sync of action item changes via WebSocket
