# S-020: Participation Metrics per Member

## User Story

As a **team lead**, I want to **see participation metrics for each team member** so that **I can understand engagement levels and ensure everyone has a voice in retrospectives**.

## Acceptance Criteria

- [x] Dashboard shows per-member metrics: cards created, votes cast, action items assigned
- [x] Metrics are aggregated across sprints with per-sprint breakdown available
- [x] Participation rate: percentage of retros a member actively participated in (submitted at least 1 card)
- [x] Engagement score per member (composite of cards, votes, action items)
- [x] Comparison visualization: bar chart or table of members ranked by engagement
- [x] Metrics respect anonymous mode (card counts shown but not card content attribution)
- [x] Data is only accessible to team admins and facilitators
- [x] Time period filter (last sprint, last 5 sprints, all time)
- [x] Export participation report as CSV
- [x] Privacy notice: members are informed that participation metrics are tracked

## Implementation Notes

- Aggregate from cards, votes, and action_items tables grouped by user
- Anonymous mode: show participation counts but never reveal which specific cards belong to whom
- Access control: only admin and facilitator roles can view individual member metrics
- Members can see their own metrics but not other individuals'
- Consider GDPR: participation data should be deletable with user account
