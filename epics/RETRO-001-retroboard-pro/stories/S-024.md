# S-024: Export Retro to JSON, Markdown, and HTML

## User Story

As a **team lead or facilitator**, I want to **export the retrospective data in JSON, Markdown, or HTML format** so that **I can share results with stakeholders, archive them, or integrate with other tools**.

## Acceptance Criteria

- [ ] Export is available for completed retro boards — Backend ✅, Frontend UI ❌ (no export button/dialog)
- [x] Three export formats are supported: JSON (raw data), Markdown (readable), HTML (formatted)
- [x] JSON export includes all board data: columns, cards, votes, groups, action items, metadata
- [x] Markdown export renders a human-readable document with headers, bullet lists, and tables
- [x] HTML export renders a styled, self-contained HTML document suitable for printing or sharing
- [x] Export includes board metadata: team name, sprint name, date, template used, participants
- [x] Cards in export are sorted by vote count (highest first) within each column
- [x] Action items section is included with status, assignee, and due date
- [x] Anonymous mode is respected in exports (no author names if anonymous)
- [ ] Export files are generated on-demand and downloaded via the browser — API works, no UI button
- [x] Export includes a summary section with basic analytics (total cards, votes, top themes)

## Implementation Notes

- Export generation happens server-side for consistency
- JSON: structured object with all entities
- Markdown: template-based generation using string interpolation
- HTML: use a Markdown-to-HTML converter + CSS styling, or a dedicated HTML template
- Return file as attachment with appropriate Content-Type and Content-Disposition headers
- Consider async generation for very large boards with a download link approach
