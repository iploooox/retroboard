# S-018: Sprint Analytics Dashboard

## User Story

As a **team lead or facilitator**, I want to **view analytics for a sprint's retrospective** so that **I can understand team sentiment, participation, and key discussion themes**.

## Acceptance Criteria

- [x] Dashboard shows total cards, votes, and action items for a sprint's retro — Note: Backend API implemented, frontend UI missing
- [x] Card distribution chart: number of cards per column (bar or pie chart) — Note: Data available via API, no frontend chart
- [x] Top voted cards are listed with their vote counts — Note: Data available via API, no frontend component
- [x] Word cloud or word frequency analysis of card content — Note: Backend endpoint `/analytics/word-cloud` exists, no frontend component
- [x] Average cards per participant metric — Note: Available in participation endpoint
- [ ] Vote distribution visualization (who voted on what) — Note: Vote data not exposed by API endpoints
- [x] Action items summary: total, completed, carried over — Note: Data included in analytics response, no frontend widget
- [ ] Dashboard is accessible to all team members (read-only) — Note: No analytics page in frontend router
- [x] Data is derived from completed retro boards
- [x] Dashboard loads within 2 seconds for sprints with up to 200 cards — Note: Backend uses materialized views for performance

## Implementation Notes

- Create materialized views or computed queries for analytics aggregations
- Consider pre-computing analytics when a board is marked as completed
- Use lightweight charting library (recharts, chart.js, or similar)
- Analytics API endpoints return aggregated data, not raw records
- Cache analytics results for completed boards (data won't change)
