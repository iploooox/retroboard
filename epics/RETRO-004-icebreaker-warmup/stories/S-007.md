# S-007: Facilitator "Start Retro" Transition with Energy Recap

## User Story

As a **facilitator**, I want a **clear "Start Retro" button that transitions everyone to the write phase simultaneously, with a brief recap of the warmup energy** so that **the transition feels deliberate, satisfying, and the team carries the warmup energy into the retro**.

## Context

The icebreaker should end with a bang, not a whimper. When the facilitator clicks "Start Retro", the icebreaker wall doesn't just disappear — there's a brief 3-second energy recap ("Your team generated 24 responses and 87 reactions!") before the board columns appear. This creates a satisfying bookend to the warmup and gives the facilitator a natural moment to say "Alright, let's go!"

## Acceptance Criteria

- [ ] Facilitator sees a prominent "Start Retro" button during icebreaker phase (fixed position, always visible)
- [ ] Non-facilitators do NOT see the "Start Retro" button (they see "Waiting for facilitator...")
- [ ] Clicking "Start Retro" triggers a confirmation (small inline confirm, not a modal — "Ready? Start Retro / Cancel")
- [ ] On confirm: server transitions board phase from `icebreaker` → `write`
- [ ] Transition is broadcast to all participants via WebSocket (`board:phase_changed`)
- [ ] Before columns appear, a brief **energy recap overlay** shows for 3 seconds:
  - "Your team is warmed up!" (headline)
  - Response count: "24 responses shared"
  - Reaction count: "87 reactions fired"
  - A burst of confetti or emoji rain during the recap (reuse S-006 rain engine)
  - Auto-dismisses after 3 seconds, or click anywhere to skip
- [ ] After recap dismisses, board columns render normally (write phase)
- [ ] If facilitator skips icebreaker (phase stepper: icebreaker → write), no recap is shown
- [ ] Energy recap data comes from the server (aggregated counts from icebreaker session)
- [ ] Recap is also visible to non-facilitator participants (everyone sees the same recap)

## Implementation Notes

### Backend

- New aggregation endpoint or include in phase transition response:
  - `GET /api/v1/boards/:boardId/icebreaker/summary`
  - Returns: `{ responseCount, reactionCount, topEmoji, participantCount }`
- Or: include the summary in the `board:phase_changed` WebSocket event payload when transitioning from icebreaker → write

### Frontend

- "Start Retro" button: large, primary color, positioned below the vibe bar or floating in bottom-right
- Inline confirmation: button transforms to "Ready? [Start Retro] [Cancel]" on first click
- Energy recap overlay: centered card with stats, confetti animation behind it
- Use S-006's `ReactionRainEngine` to fire a celebration burst (mix of all 6 emojis, large burst)
- Auto-dismiss: `setTimeout(3000)` + click-anywhere handler
- After dismiss: set `showRecap = false`, board renders normally in write phase

### Recap Design

```
┌────────────────────────────────┐
│                                │
│     🎉 Your team is           │
│        warmed up!              │
│                                │
│   24 responses shared          │
│   87 reactions fired           │
│   🔥 was the vibe             │
│                                │
│        [Let's go!]             │
│                                │
└────────────────────────────────┘
      (emojis raining behind)
```
