# S-001: Icebreaker as a Facilitator-Controlled Board Phase

## User Story

As a **facilitator**, I want the **icebreaker to be a proper board phase that I control for the whole team** so that **everyone participates in the warmup together instead of dismissing it individually**.

## Context

Today the icebreaker is per-user client state (`useState(true)`) — each person dismisses it on their own. There's no shared experience. This story makes the icebreaker a first-class board phase, stored in the database and controlled by the facilitator like every other phase.

## Acceptance Criteria

- [ ] `board_phase` enum includes `icebreaker` as a valid phase (before `write`)
- [ ] New boards start in `icebreaker` phase by default (configurable — see S-008)
- [ ] Phase transition order: `icebreaker` → `write` → `group` → `vote` → `discuss` → `action`
- [ ] Facilitator can skip icebreaker (`icebreaker` → `write` directly) via phase controls
- [ ] Facilitator can go back: `write` → `icebreaker` (in case they skipped too fast)
- [ ] Phase transition from `icebreaker` to `write` is broadcast to all participants via WebSocket
- [ ] Non-facilitator participants cannot advance past the icebreaker phase
- [ ] During icebreaker phase: card creation, voting, grouping are all disabled (server-enforced)
- [ ] The existing phase stepper UI shows icebreaker as the first step
- [ ] `IcebreakerCard.tsx` is replaced — icebreaker phase renders a new `IcebreakerWarmup` component (the wall)
- [ ] Old `showIcebreaker` local state is removed from `BoardPage.tsx`

## Implementation Notes

### Database

- Migration: `ALTER TYPE board_phase ADD VALUE 'icebreaker' BEFORE 'write'`
- Add `icebreaker_id` column to `boards` table (nullable UUID FK → `icebreakers.id`) — stores the active question
- Add `icebreaker_active` boolean column to `boards` (default `true`) — team can disable entirely (S-008)
- Update `boards.phase` default from `'write'` to `'icebreaker'`

### Backend

- Update `BOARD_PHASES` array in `validation/boards.ts`: add `'icebreaker'` at index 0
- Update `ALLOWED_TRANSITIONS`: `icebreaker → [write]`, `write → [icebreaker, group]`
- Update phase validation middleware — icebreaker phase blocks all card/vote/group operations
- When board is created: if team has icebreaker enabled, start in `icebreaker` phase; otherwise start in `write`
- Add `boards.icebreaker_id` to board GET response

### Frontend

- Replace the `showIcebreaker` pattern in `BoardPage.tsx` with phase-based rendering
- When `board.phase === 'icebreaker'`, render `<IcebreakerWarmup>` instead of board columns
- Phase stepper shows: Icebreaker → Write → Group → Vote → Discuss → Action
- Icebreaker step in stepper uses a distinct icon (dice/sparkle)
