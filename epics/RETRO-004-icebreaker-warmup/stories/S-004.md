# S-004: Response Wall Layout and Card Animations

## User Story

As a **team member**, I want to **see responses appear on the wall with satisfying animations** so that **the warmup feels alive, dynamic, and fun ‚Äî not like a static list**.

## Context

The wall is the centerpiece of the icebreaker. If responses just "pop in" like a list, it's boring. If they float in with staggered animations, random slight rotations, and soft shadows, it feels like sticky notes being placed on a real wall. The animation creates delight and anticipation ‚Äî you *want* to see the next response appear.

## Acceptance Criteria

- [ ] Response cards appear with a staggered entrance animation (not all at once on load)
- [ ] New real-time responses animate in with a "place on wall" effect (scale up from 0.8 ‚Üí 1.0 + fade in)
- [ ] Each card has a subtle random rotation (-2deg to +2deg) ‚Äî like real sticky notes on a wall
- [ ] Cards have slightly varied background colors (soft pastels) ‚Äî not all the same white
- [ ] Wall layout is a masonry-style or wrapped grid ‚Äî not a boring vertical list
- [ ] Cards have a subtle hover effect (lift + shadow increase) to feel tactile
- [ ] When a card is deleted by facilitator, it fades out with a shrink animation
- [ ] On initial load, existing responses animate in with a cascade (50ms delay between each)
- [ ] Wall auto-scrolls to show newest responses when they arrive (smooth scroll)
- [ ] Layout is responsive: 3-4 columns on desktop, 2 on tablet, 1 on mobile
- [ ] Animations are CSS-only (no JS animation libraries) ‚Äî `@keyframes` + `transition`
- [ ] Animations respect `prefers-reduced-motion` ‚Äî users with motion sensitivity see instant appearance

Assign color deterministically based on response ID hash (consistent across reloads).

### Animation Keyframes

```css
@keyframes cardEntrance {
  from {
    opacity: 0;
    transform: scale(0.85) translateY(10px);
  }
  to {
    opacity: 1;
    transform: scale(1) translateY(0) rotate(var(--card-rotation));
  }
}

@keyframes cardExit {
  to {
    opacity: 0;
    transform: scale(0.8);
  }
}
```

### Performance

- Use `will-change: transform, opacity` on animating cards
- Remove `will-change` after animation completes (via `animationend` event)
- CSS containment (`contain: layout style`) on wall container
- Limit wall to last 50 responses (older ones still in DB, just not rendered)
