# S-006: Vibe Bar with Reaction Rain Animation

## User Story

As a **team member**, I want to **fire off emoji reactions from a persistent vibe bar and see them rain across everyone's screen** so that **the icebreaker builds visible energy, feels like a live event, and makes people smile**.

## Context

This is the signature feature. The Vibe Bar is a persistent emoji bar at the bottom of the icebreaker screen. Tap any emoji — it doesn't attach to a specific response, it fires off across the whole screen. When 5 people tap 🔥 within 2 seconds, a burst of fire emojis floods the screen. Think Twitch reactions, Instagram Live hearts, concert crowd.

This is separate from S-005 (per-response reactions). S-005 = "I like this specific answer." S-006 = "I'm feeling the vibes." Both coexist.

## Acceptance Criteria

- [ ] A Vibe Bar is fixed at the bottom of the icebreaker screen (above the response input from S-003)
- [ ] Vibe Bar shows the same 6 emojis: 😂 🔥 ❤️ 🎯 👏 💀
- [ ] Tapping an emoji in the Vibe Bar fires a "vibe reaction" — purely visual, not tied to any response
- [ ] The emoji animates across **everyone's screen** in real-time via WebSocket
- [ ] Each vibe reaction is a single emoji that floats/falls/bounces across the viewport:
  - 😂 bounces in from the bottom with wobble
  - 🔥 rains down from the top with a subtle glow
  - ❤️ floats upward gently (like Instagram Live hearts)
  - 🎯 zooms in from center, pulses, fades
  - 👏 cascades in from the sides
  - 💀 tumbles down with rotation
- [ ] Multiple emojis on screen simultaneously — up to 30 active at once (older ones fade out)
- [ ] Burst detection: when 3+ of the same emoji arrive within 2 seconds, trigger a "burst" — 5x more emojis with bigger size
- [ ] Vibe reactions are rate-limited: max 3 per second per user (prevents spam, still feels responsive)
- [ ] Vibe reactions are ephemeral — NOT stored in the database. Fire and forget. They're visual-only
- [ ] Animation renders at 60fps with 20+ concurrent emojis
- [ ] Vibe Bar has a subtle pulse/glow when reactions are flying (visual feedback that things are active)
- [ ] Animations respect `prefers-reduced-motion` — reduced to simple fade in/out

## Implementation Notes

### Architecture

Vibe reactions are ephemeral — they exist only as WebSocket messages. No database table. No persistence.

```
User taps 🔥 → POST /boards/:boardId/icebreaker/vibe (or WS message)
           → Server broadcasts to all board participants
           → Each client renders the animation locally
           → Animation ends, emoji is garbage collected
```

For performance, vibe reactions should be sent as WebSocket messages directly (not REST + broadcast). This avoids HTTP overhead for something that happens rapidly.

### WebSocket Protocol

Client sends:
```json
{ "type": "icebreaker_vibe", "payload": { "emoji": "fire" } }
```

Server broadcasts to all in room (including sender):
```json
{ "type": "icebreaker_vibe", "payload": { "emoji": "fire", "id": "uuid" } }
```

The `id` is server-generated to deduplicate. Server rate-limits: 3 per second per user.

### Animation Engine (Frontend)

Create a lightweight `ReactionRainEngine`:

1. **Pool**: Pre-create 30 DOM elements (emoji containers), reuse them (object pool pattern)
2. **Spawn**: When a vibe event arrives, grab an idle element from the pool, set its emoji + animation class, add to viewport
3. **Animate**: Each emoji type has a CSS `@keyframes` animation (1.5-3s duration)
4. **Recycle**: On `animationend`, reset the element and return to pool
5. **Burst**: If 3+ same emoji in 2s window, spawn 5 extra with larger size class

```css
@keyframes rainFire {
  0%   { top: -40px; opacity: 0; transform: scale(0.5); }
  10%  { opacity: 1; transform: scale(1); }
  90%  { opacity: 0.8; }
  100% { top: 100vh; opacity: 0; transform: scale(0.3); }
}

@keyframes floatHeart {
  0%   { bottom: -40px; opacity: 0; transform: scale(0.5) translateX(0); }
  20%  { opacity: 1; transform: scale(1); }
  100% { bottom: 100vh; opacity: 0; transform: scale(0.6) translateX(30px); }
}
```

### Vibe Bar Design

```
┌──────────────────────────────────────────────┐
│  😂    🔥    ❤️    🎯    👏    💀           │  ← Fixed bottom bar
└──────────────────────────────────────────────┘
```

- Emojis are large (32px), spaced evenly, tappable area is generous (48px hit target)
- Subtle glass-morphism background (`backdrop-blur-sm bg-white/80`)
- When reactions are active, the bar has a subtle pulse glow animation
- Tap feedback: button scales down briefly (0.9) on press

### Performance

- Use CSS `transform` and `opacity` only (GPU-composited, no layout thrash)
- Object pool of 30 elements — never create/destroy DOM nodes during animation
- `will-change: transform, opacity` on pool elements
- `requestAnimationFrame` for burst detection timer, not `setInterval`
- `pointer-events: none` on animated emojis (don't block clicks)
