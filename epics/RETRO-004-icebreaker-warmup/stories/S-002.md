# S-002: Shared Question Display with Facilitator Picker

## User Story

As a **facilitator**, I want to **pick, reroll, or filter icebreaker questions that everyone sees simultaneously** so that **I can choose something appropriate for my team's mood and everyone discusses the same question**.

## Context

Today each user fetches their own random question independently — two people can see completely different questions. This story makes question selection a facilitator-only action that's broadcast to everyone in real-time.

## Acceptance Criteria

- [ ] When board enters icebreaker phase, a random question is auto-selected and stored as `boards.icebreaker_id`
- [ ] The selected question is displayed to ALL participants — same question, same time
- [ ] Only the facilitator sees the question controls (reroll, category filter)
- [ ] Facilitator can reroll → new random question is selected, stored in DB, broadcast via WebSocket to everyone
- [ ] Facilitator can filter by category → reroll pulls from that category only
- [ ] Non-facilitators see the question and category badge but no controls (read-only)
- [ ] Question selection records usage in `team_icebreaker_history` (prevents repeats across retros)
- [ ] The question display is large, centered, and prominent — the focal point of the screen
- [ ] Category badge is visible below the question (e.g., "Fun", "Team-Building")
- [ ] Loading state shown while question is being fetched

## Implementation Notes

### Backend

- New endpoint: `PATCH /api/v1/boards/:boardId/icebreaker` — sets `icebreaker_id`, broadcasts, records history
  - Only facilitator/admin can call this
  - Accepts optional `category` filter
  - Calls existing `icebreakerService.getRandom()` logic
  - Updates `boards.icebreaker_id`
  - Records in `team_icebreaker_history`
  - Broadcasts `icebreaker_question_changed` via WebSocket with `{ id, question, category }`
- On board creation: auto-select a random icebreaker and set `boards.icebreaker_id`
- Board GET response includes the full icebreaker object (question text + category), not just the ID

### Frontend

- `IcebreakerWarmup` component renders the question from board state (not local fetch)
- Facilitator controls: category filter pills + "New Question" button (reuse existing category UI)
- Non-facilitator: question + category badge only
- WebSocket listener for `icebreaker_question_changed` → update board store → UI re-renders
- Question text: `text-3xl font-bold` centered, with subtle fade-in transition on change

### Reuse from S-028

- `icebreakerService.getRandom()` — works as-is with category filter and team history exclusion
- `icebreakerService.recordHistory()` — works as-is
- Category constants — reuse `CATEGORIES` array
- The 55 system questions and category filter UI design
